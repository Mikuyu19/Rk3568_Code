# MCU最小系统与PCB设计笔记

## 1，嵌入式系统介绍

​	嵌入式系统是以应用为中心，以计算机技术(并融入微电子技术、通信技术和自动化控制技术)为基础，软硬件可以裁剪，适用于对功能、成本、体积、功耗有严格要求的**专用计算机系统**。

```
计算机系统分类：
	通用计算机系统：
		是相对于嵌入式计算机系统而言的，需兼顾多种任务，综合性能强，软件生态丰富，适用性广，但硬件成本较高。如：PC(个人电脑)、服务器。
	专用计算机系统：
		(嵌入式系统)主要由嵌入式微控制器(MCU)、外围硬件装置、嵌入式操作系统以及使用者应用软件等部分组成。为特定任务设计，硬件和软件高度定制。它具有“嵌入式”、“专用性”和“计算机系统”三个基本要素。
不同点：
	1，嵌入式的形态随产品设备不同而不同(把专用计算机系统嵌入到不同的产品)
	2，功能专用性
	3，功耗低
	4，价值体现在所嵌入和控制的设备上。比如：洗衣机的价值体现在性价比、洗净程度、是否节能、洗涤容量等。通用计算机的价值体现在“计算性能”和“存储空间”等方面。
```

嵌入式系统：软硬件结合 ==》 软件+硬件的综合体，那么什么是软件、什么是硬件呢？

```
1）硬件：是指计算机的实体。我们能看到的各种电子元器件。各类光、电、机械设备的实物组成。如：键盘、鼠标。。
2）软件：是我们看不见的部分，由人们实现编制的各种特殊功能的程序组成。这些程序保存在各种存储设备(如：ROM、RAM、硬盘、磁盘、光盘等)。软件又分为系统软件和应用软件。
```

## 2，MCU最小系统讲解

​	MCU最小系统电路是指微控制器单元(Micro Controller Unit, MCU)能够正常运行所需的最基本电路配置。它包含了确保MCU稳定工作的核心电路模块，是嵌入式系统设计的起点和基础。

​	MCU最小系统电路一般由 MCU芯片、电源电路、复位电路、时钟电路、程序下载和调试接口电路组成。各部分的功能作用如下：

```
1，MCU芯片
	MCU(Micro Controller Unit),又称单片机或单芯片微型计算机，以高集成度、低功耗、专用性强等特征，是嵌入式系统的核心控制单元。常见的MCU有 C51系列单片机、STM32系列单片机、ESP32等。
2，电源电路
作用: 为MCU及其外围电路提供稳定的工作电压，确保MCU能够持续、稳定的运行。
细节：电源电路将外部电源(如电池或适配器)转换为MCU所需的电压，通常为 3.3V或5V。
	电源电路中常配有去耦电容(如100nF、10uF),用于滤除电源噪声、保证电源质量。
	对于复杂MCU，可能有多个电源引脚(如VDD,VDDA,VBAT等)，需根据MCU规格书设计电源网络。
3，复位电路
作用：在系统上电或运行过程中出现异常时，将MCU内部状态恢复至初始状态，确保程序从头开始执行。
细节：复位电路通常包括上拉电阻和滤波电容，上拉电阻保证MCU正常工作时复位引脚为高电平。
	按键复位或外部信号复位时，复位引脚被拉至低电平，触发MCU复位。
	一些MCU支持多种复位方式(如软件复位、看门狗复位等)，提供系统可靠性。
4，时钟电路
作用：为MCU提供稳定的时钟信号；确保其能够按照预定的节奏和时序进行工作。
细节：时钟信号通常由外部晶振(如12MHz,8MHz)或内部时钟源(如RC振荡器)提供。
	外部晶振需要配合电容使用，以提供时钟信号的稳定性。时钟电路的设计直接影响MCU的运行速度和性能。
5，程序下载和调试接口电路
作用：用于将编写好的程序下载到MCU中，并进行调试和监控。
细节：常见的下载和调试接口包括 UART、JTAG和SWD等。
	UART接口通过串口进行下载和调试，简单方便。
	JTAG和SWD接口提供更快的下载速度和更强大的调试功能，支持实时跟踪和断点调试。
```

### 3.3 外部DC输入电路设计

​	供电输入常见有4种方式：适配器供电、外接12V锂电池包供电、外接电池盒供电、USB供电。

​	电脑的USB的输入电压为5V，且市面上大部分项目能用到的模块(比如MQ-2烟雾模块、GY39)的电压为5V。

​	所有如果用适配器或者外接12V锂电池包供电时，需要把电压降压到 5V。我们做小车项目时，电机的驱动电压一般为 7.2V~12V。也就是说我既要实现 7.2V~12V 降压到 5V的电源输出电路，同时也保留提供 7.2 V~12V的输出。

### 3.9 串口模块接口电路设计

​	串口通信协议是最常用也是最简单的通信协议，是 STM32非常重要的一个知识点。有很多常见的传感器模块和通信模块都是用串口协议进行通信，比如 蓝牙模块(HC-05)、WIFI模块(ESP8266)、GY39等。

​	STM32F103RCT6是入门级MCU，所以串口资源有限，我们需要合理的分配串口资源：

-   串口程序下载 ==> 串口1(固定的没有办法改变的，因为只有串口1才能烧录程序)
-   蓝牙模块 ==> 串口2
-   WIFI模块 ==> 串口3

串口模块的使用流程如下：

1，用跳线帽来切换 串口的连接线路；

2，模块连接电脑通过 CH340电路，使用PC端的串口助手来进行调试；

3，编写串口通信程序，MCU通过串口来与模块之间进行通信。

**注意：**ESP8266 WIFI模块属于高功率的器件，尤其是收发的时候，峰值功率特别高，所有需要进行单独的供电。

### 3.10 ADC模块电路设计(AD模数转换：模拟信号 ==> 数字信号)

​	STM32 中 ADC 的输入的电压范围为 0~3.3V。但是市面上的 ADC 模块输出电压范围基本分为两种：

1）输出模拟信号范围为:0~3.3V, 可以直接接入到 ADC 输入通道。

2）输出模拟信号范围为: 0~5V, 不可以直接接到 ADC 输入通道，需要进行分压。

我们实现3路ADC采集电路。

### 4.4 PCB布局

#### 4.41 元件排列规则

```
1，在通常的条件下，所有的元件均应布置在PCB的同一面上，只有在顶层元件过密时，才能将一些高度有限并发热量小的元件(如贴片电阻、贴片电容、贴片IC等)放在底层。
2，在保证电气性能的前提下，元件应放置在栅格上且相互平行或垂直排列，以求整齐、美观，尽量不要出现线路交叉。一般情况下不允许元件堆叠，元件排列要紧凑、输入元件和输出元件尽量分开远离。
3，某些元件或导线之间可能存在较高电压，应加大它们的距离，以免因放电、击穿而引起意外的短路，布局的时候应用尽可能注意这些信号的布局空间。
4，用于外供电源的接口一般统一把VCC摆左边，GND摆右边，防止反接。带高压的元件尽量布置在调试时手不容易触及的地方。
5，一般把需要经常插拔的接口放在板子边缘。且位于板边缘的元件，应该尽量做到离板边缘有两个板厚的距离。
6，元件在整个板面上应分布均匀，不要这一块区域密，另一块区域疏松，提高产品的可靠性。
```

#### 4.4.2 按照信号走向布局原则

```
1，放置固定元件之后，按照信号的流向逐个安排各个功能电路单元的位置，以及每个功能单元的核心元件为中心，围绕他进行局部布局。
2，元件的布局应便于信号流通，使信号尽量可能保持一致的方向。在多数情况下，信号的流向安排从左到右或从上到下，与输入 、输出端直接相连的元件应当放在靠近输入、输出接插件或连接件的地方。
```

#### 4.4.3 STM32F103RCT6项目用开发板 PCB 布局参考

1）先设置好四个角落的安装孔，用于将板子安装在设备中，比如通过螺丝安装到小车上。

​	在放置(P)功能中找到焊盘(P),设置通用直径为4mm，钻孔属性直径为 3.5mm。

​	把四个安装孔的圆心离边界3mm，属性里的网络设置为 GND(可以增强电路稳定性)。在位置中设置四个安装孔的坐标(中心x,中心y)为如下：

```
0:  3mm,	-3mm
1:	97mm,	-3mm
2:	97mm,	-97mm
3:	3mm,	-97mm
```

2) 电路布局先后顺序思路：把核心板摆放中间，然后先布局板子边缘电路，再填充中间的电路。

```
电路布局顺序：
1）核心板电路(放中央)
2）外部DC输入电路(左下角)
3）5V-3.3V转换电路(左中边缘)以及电源输出电路(左上角)
4）USB&串口下载调试供电电路(右下边缘)
5）按键电路(右下角)
6）蜂鸣器、LED(插空)
7）蓝牙接口和舵机接口电路(左插空)
8）ADC接口电路(保留20mm宽度)(上边缘)
9）OLED接口电路(右中边缘)
10）ESP8266接口电路(保留15cm宽度)(右上角)
11）SPI-Flash(右中插空)
12）串口切换电路(右中插空)
```

3）PCB布局细节技巧

(1) 元器件转换到PCB的排列是按照原理图的位置来的。如果不知道器件在原理图的什么位置。可以选择元件后点击 **交叉选择**，即可指定位置。

(2) 利用布局(O)中的 **对齐** 和 **分布** 使元件摆放整齐。注意元器件是否摆放整齐是后续能够自动化布线成功的关键因素。

(3) 可以在文档层 用 测量工具 刻画尺度。

### 4.5 PCB 布线

​	布线分为自动布线、手动布线。初学者建议使用自动布线再手动调整。

​	自动布线步骤如下：

​	在设计规则中把 VCC相关的导线(+5V,VCC,VCC-EX,VCC-MCU)都加粗。

​	1）布线 ==> 自动布线 ==> 设计规则

​	2）物理 ==> 导线 ==> 新增 power wire

​		单位换成mil，最小20，默认25，最大45

​	3）网络规则 ==> 导线：把 VCC相关的导线(+5V,VCC,VCC-EX,VCC-MCU)设置为 power wire

​	4）运行 ==> 网络布通率必须是 100%

如果没有成功 ==> 清除全部网络，调整后再次自动布线
