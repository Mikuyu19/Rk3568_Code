## 3. 库文件 / Makefile / CMake 相关指令

1. 动态库 (Shared Library, .so) 
核心原理：运行期加载，多程序共享，极大地节省开发板内存和磁盘空间。

```bash
# 1. 将多个源文件编译为动态库 (必须加 -fPIC 和 -shared)
# 规范：输出文件必须以 lib 开头，以 .so 结尾
gcc -shared -fPIC bst.c linkqueue.c -o libbst.so

# 2. 编译主程序并链接动态库
# -I ./ : 告诉编译器去当前目录找头文件 (如 bst.h)
# -L ./ : 告诉链接器去当前目录找库文件
# -l bst : 链接库，掐头去尾 (去lib，去.so)
gcc main.c -I ./ -L ./ -l bst -o app

# 3. 运行前设置环境变量 (给操作系统动态加载器指路)
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:.
./app

```



2. 静态库 (Static Library, .a)
核心原理：编译期直接将机器码拷贝进可执行文件中，体积大，但不依赖外界环境。

```
# 1. 先生成目标文件 (.o)
gcc -c bst.c linkqueue.c

# 2. 使用 ar 工具打包成静态库 (必须以 lib 开头，.a 结尾)
ar rcs libbst.a bst.o linkqueue.o

# 3. 链接主程序 (命令与动态库完全一样，GCC优先链接同名动态库)
gcc main.c -I ./ -L ./ -l bst -o app_static
```

  3.Makefile
        核心痛点解决：替代手动敲击 gcc 的繁琐过程，实现增量编译（只编译修改过的文件）。

```makefile

# ==========================================
# 1. 变量定义区 (方便一键切换为交叉编译)
# ==========================================
# 如果要在开发板跑，把 gcc 改成 aarch64-linux-gnu-gcc
CC = gcc  
CFLAGS = -I./ -fPIC -Wall
LDFLAGS = -L./ -lbst

# ==========================================
# 2. 终极目标区
# ==========================================
all: libbst.so app

# ==========================================
# 3. 规则定义区
# 魔法变量速查：
# $@ : 代表目标文件 (冒号左边)
# $^ : 代表所有依赖文件 (冒号右边)
# $< : 代表第一个依赖文件
# ==========================================

# 目标：生成动态库
libbst.so: bst.c linkqueue.c
	$(CC) -shared $(CFLAGS) -o $@ $^

# 目标：生成可执行程序
app: main.c libbst.so
	$(CC) $< $(CFLAGS) $(LDFLAGS) -o $@

# ==========================================
# 4. 伪目标与清理
# ==========================================
.PHONY: clean
clean:
	rm -f app libbst.so *.o
```

1. 核心 CMakeLists.txt 模板
  无论工程怎么变，最核心的骨架只有这 6 句代码

```cmake
# 1. 设定版本与项目名称
cmake_minimum_required(VERSION 3.10)
project(RK3568_Project)

# 2. 指定头文件搜索路径 (等价于 -I)
include_directories(${CMAKE_CURRENT_SOURCE_DIR})

# 3. 生成动态库 (等价于 gcc -shared -fPIC...)
# CMake 会自动补充 lib 前缀和 .so 后缀
add_library(bst SHARED bst.c linkqueue.c)

# 4. 生成可执行文件
add_executable(app main.c)

# 5. 链接依赖库 (等价于 -L 和 -l)
# CMake 非常聪明，它知道 bst 在哪，全自动处理链接路径
target_link_libraries(app bst)
```
