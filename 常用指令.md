## 2.Linux常用指令

1. 文件与目录

   | **指令**    | **常用参数**                                  | **说明**                         |
   | ----------- | --------------------------------------------- | -------------------------------- |
   | **`ls`**    | `-l` (详细), `-a` (显示隐藏), `-h` (易读大小) | 列出目录内容                     |
   | **`cd`**    | `..` (上级), `~` (主目录), `-` (上次目录)     | 切换工作目录                     |
   | **`pwd`**   | 无                                            | 显示当前完整路径                 |
   | **`mkdir`** | `-p` (递归创建多级目录)                       | 创建新目录                       |
   | **`rm`**    | `-r` (递归删除目录), `-f` (强制删除)          | 删除文件或目录（**慎用 `-rf`**） |
   | **`cp`**    | `-r` (拷贝目录), `-p` (保留属性)              | 复制文件或目录                   |
   | **`mv`**    | 无                                            | 移动或重命名文件/目录            |
   | **`touch`** | 无                                            | 创建空文件或更新文件时间戳       |

2. 文件内容查看

   **`cat`**: 一次性显示文件全部内容。

   **`less`**: 分页查看长文件（支持上下滚动，按 `q` 退出）。

   **`head` / `tail`**:

   - `tail -n 20 file`: 查看最后 20 行。
   - **`tail -f file`**: **实时刷新**查看文件末尾（常用于看日志）。

   **`grep`**: 强大的文本搜索工具。

   - `grep "error" log.txt`: 在文件中搜索关键词。
   - `ps -ef | grep python`: 结合管道符过滤进程。

   **`find`**: 在文件系统中查找文件。

   - `find /path -name "*.conf"`: 按文件名搜索。

3. 系统状态与进程

   **`top` / `htop`**: 实时显示系统资源占用（CPU、内存）及进程列表。

   **`ps`**: 静态查看进程快照。

   - `ps -ef` 或 `ps aux`: 显示所有运行中的进程。

   **`kill`**: 终止进程。

   - `kill -9 <PID>`: 强制杀死指定进程。

   **`df`**: 查看磁盘剩余空间。

   - `df -h`: 以 GB/MB 格式显示。

   **`free`**: 查看内存使用情况。

   - `free -h`: 易读格式显示。

4. 网络相关

   **`ping`**: 测试网络连通性。

   **`ifconfig`** (或 `ip addr`): 查看本机的 IP 地址和网卡信息。

   **`netstat`**: 查看网络端口占用。

   - `netstat -tuln`: 查看当前系统监听的端口。

   **`curl` / `wget`**:

   - `curl -X GET http://example.com`: 发送网络请求。
   - `wget http://url/file.zip`: 下载文件。

5. 权限相关

   **`chmod`**: 修改文件权限。

   - `chmod 755 file`: 设置为 rwxr-xr-x。
   - `chmod +x script.sh`: 赋予执行权限。

   **`chown`**: 修改文件所有者。

   - `chown user:group file`: 更改属主和属组。

   **`sudo`**: 以超级用户（root）身份执行命令。

6. 压缩与归档

   **`tar`**:

   - 解压：`tar -zxvf file.tar.gz`
   - 压缩：`tar -zcvf output.tar.gz /path/to/dir`

   **`zip` / `unzip`**: 处理 .zip 格式文件。

## 4. 库文件 / Makefile / CMake 相关指令

1. 动态库 (Shared Library, .so) 
    核心原理：运行期加载，多程序共享，极大地节省开发板内存和磁盘空间。

  ```makefile
  # 1. 将多个源文件编译为动态库 (必须加 -fPIC 和 -shared)
  # 规范：输出文件必须以 lib 开头，以 .so 结尾
  gcc -shared -fPIC bst.c linkqueue.c -o libbst.so
  
  # 2. 编译主程序并链接动态库
  # -I ./ : 告诉编译器去当前目录找头文件 (如 bst.h)
  # -L ./ : 告诉链接器去当前目录找库文件
  # -l bst : 链接库，掐头去尾 (去lib，去.so)
  gcc main.c -I ./ -L ./ -l bst -o app
  
  # 3. 运行前设置环境变量 (给操作系统动态加载器指路)
  export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:.
  ./app
  ```

  

2. 静态库 (Static Library, .a)
    核心原理：编译期直接将机器码拷贝进可执行文件中，体积大，但不依赖外界环境。

  ```makefile
  # 1. 先生成目标文件 (.o)
  gcc -c bst.c linkqueue.c
  
  # 2. 使用 ar 工具打包成静态库 (必须以 lib 开头，.a 结尾)
  ar rcs libbst.a bst.o linkqueue.o
  
  # 3. 链接主程序 (命令与动态库完全一样，GCC优先链接同名动态库)
  gcc main.c -I ./ -L ./ -l bst -o app_static
  ```

  

3. makefile

  核心痛点解决：替代手动敲击 gcc 的繁琐过程，实现增量编译（只编译修改过的文件）。

  ```makefile
  # ==========================================
  # 1. 变量定义区 (方便一键切换为交叉编译)
  # ==========================================
  # 如果要在开发板跑，把 gcc 改成 aarch64-linux-gnu-gcc
  CC = gcc  
  CFLAGS = -I./ -fPIC -Wall
  LDFLAGS = -L./ -lbst
  
  # ==========================================
  # 2. 终极目标区
  # ==========================================
  all: libbst.so app
  
  # ==========================================
  # 3. 规则定义区
  # 魔法变量速查：
  # $@ : 代表目标文件 (冒号左边)
  # $^ : 代表所有依赖文件 (冒号右边)
  # $< : 代表第一个依赖文件
  # ==========================================
  
  # 目标：生成动态库
  libbst.so: bst.c linkqueue.c
  	$(CC) -shared $(CFLAGS) -o $@ $^
  
  # 目标：生成可执行程序
  app: main.c libbst.so
  	$(CC) $< $(CFLAGS) $(LDFLAGS) -o $@
  
  # ==========================================
  # 4. 伪目标与清理
  # ==========================================
  .PHONY: clean
  clean:
  	rm -f app libbst.so *.o
  ```

  



1. 核心 CMakeLists.txt 模板
    无论工程怎么变，最核心的骨架只有这 6 句代码
    
    ```cmake
    # 1. 设定版本与项目名称
    cmake_minimum_required(VERSION 3.10)
    project(RK3568_Project)
    
    # 2. 指定头文件搜索路径 (等价于 -I)
    include_directories(${CMAKE_CURRENT_SOURCE_DIR})
    
    # 3. 生成动态库 (等价于 gcc -shared -fPIC...)
    # CMake 会自动补充 lib 前缀和 .so 后缀
    add_library(bst SHARED bst.c linkqueue.c)
    
    # 4. 生成可执行文件
    add_executable(app main.c)
    
    # 5. 链接依赖库 (等价于 -L 和 -l)
    # CMake 非常聪明，它知道 bst 在哪，全自动处理链接路径
    target_link_libraries(app bst)
    ```

4. 内置函数

   ```makefile
   1. wildcard（通配符自动扫描）
   作用：像雷达一样，去指定的目录扫描所有符合规则的文件名，并用空格隔开。
   语法：$(wildcard 匹配模式)
   
   # 获取当前目录下所有的 .c 文件，存入变量 SRC
   SRC = $(wildcard *.c)
   
   2. patsubst（模式批量替换）
   作用：把一串文本中符合条件的单词，批量替换成你想要的格式（常用于把 .c 列表映射为 .o 列表）。
   语法：$(patsubst 查找模式, 替换模式, 目标文本串)
   # 把 SRC 变量里所有的 .c 替换成 .o，存入变量 OBJ
   OBJ = $(patsubst %.c, %.o, $(SRC))
   
   3. shell（执行外部系统命令）—— [高频补充]
   作用：在 Makefile 里直接运行 Linux 终端命令，并把结果拿回来。
   语法：$(shell 系统命令)
   # 获取当前所在的绝对路径
   CUR_DIR = $(shell pwd)
   # 获取系统的架构信息 (用于判断是不是 aarch64)
   ARCH = $(shell uname -m)
   
   4. notdir（剥离目录路径）—— [高频补充]
   作用：当你拿到的文件名带有很长的路径时，用它可以只保留最后的文件名。
   语法：$(notdir 包含路径的文件名列表)
   # 去掉 src/ 路径，只保留文件名
   FILES = $(notdir src/main.c src/add.c)
   # 此时 FILES 的值就是 "main.c add.c"
   ```

   

## 5.Makefile万能模板

1. Makefile万能模板

   ```makefile
   # 1. 编译器与目标名称定义
   # 【注意】如果你要放到 RK3568 开发板上跑，请将 gcc 改为 aarch64-linux-gnu-gcc
   CC = gcc
   #CC = aarch64-linux-gnu-gcc
   TARGET = app
   
   # 2. 编译选项与链接路径配置
   CFLAGS = -I./ -Wall -g      # -I 指定头文件路径，-Wall 开启警告，-g 支持GDB调试
   LDFLAGS = -L./              # -L 指定库文件路径 (如果需要链动态库，在后面加 -lxxx)
   LIBname = -l
   
   # 3. 魔法函数：自动处理源文件
   # 扫描当前目录下所有的 .c 文件
   SRC = $(wildcard *.c)
   # 将所有的 .c 文件名映射为对应的 .o 文件名
   OBJ = $(patsubst %.c, %.o, $(SRC))
   
   # 4. 终极编译规则 (链接成可执行文件)
   $(TARGET): $(OBJ)
   	$(CC) $^ -o $@ $(LDFLAGS)
   
   # 5. 模式编译规则 (教 make 如何把 .c 编译成 .o)
   %.o: %.c
   	$(CC) -c $< -o $@ $(CFLAGS)
   
   # 6. 防坑清理规则
   .PHONY: clean
   clean:
   	rm -f $(TARGET) $(OBJ)
   ```

   

2. Cmake万能模板

   ```cmake
   # 1. 指定 CMake 的最低版本要求
   cmake_minimum_required(VERSION 3.10)
   
   # 2. 设置编译路径（指定工具链）
   set(CMAKE_C_COMPILER /usr/local/arm/gcc-linaro-7.5.0-2019.12-x86_64_aarch64-linux-gnu/bin/aarch64-linux-gnu-gcc)
   set(CMAKE_CXX_COMPILER /usr/local/arm/gcc-linaro-7.5.0-2019.12-x86_64_aarch64-linux-gnu/bin/aarch64-linux-gnu-g++)
   
   # 2. 定义项目名称 (后续可以通过 ${PROJECT_NAME} 变量来引用它)
   project(RK3568_App)
   
   # 3. 设置全局编译选项 (相当于 Makefile 里的 CFLAGS)
   # 这里开启了所有警告，并加入了 GDB 调试信息支持
   add_compile_options(-Wall -g)
   
   # 4. 指定头文件的搜索目录 (相当于 Makefile 里的 -I ./)
   include_directories(${CMAKE_CURRENT_SOURCE_DIR})
   
   # 5. 魔法扫描：自动查找当前目录下所有的源码文件，并存入变量 DIR_SRCS 中
   aux_source_directory(. DIR_SRCS)
   
   # 6. 指定库文件所在的路径
   link_directories(./lib)
   
   # 7. 添加lib子目录
   add_subdirectory(./lib)
   
   # 8. 生成可执行文件
   # 使用上一步扫描到的所有源码，生成以项目名字命名的可执行文件
   add_executable(${PROJECT_NAME} ${DIR_SRCS})
   
   # 9. 链接依赖的动态库/静态库 (相当于 Makefile 里的 -L 和 -l)
   # 【注意】如果你的项目不需要链接额外的库，这一行可以直接注释掉
   target_link_libraries(${PROJECT_NAME} pthread m)
   ```

## 1.文件IO

1. **open**

   ```c
   #include <sys/types.h>
   #include <sys/stat.h>
   #include <fcntl.h>
   
   int open(const char *pathname, int flags);//打开一个已经存在的文件
   int open(const char *pathname, int flags, mode_t mode);//创建一个新的文件并打开
   参数：
   	pathname：要打开的文件或创建文件的路径(绝对路径或相对路径)
   	flags：打开文件标记
   		O_RDONLY只读  0
   		O_WRONLY只写	1
   		O_RDWR可读可写	2
   		注意：上面的三个状态标记只能选择其中的一个
   		O_CREAT文件如果不存在，则创建一个文件
   		O_TRUNC文件已经存在，则进行截短(变成空的文件)
   		O_APPEND追加，设置文件的偏移量会在文件末尾(写)
   		O_NONBLOCK以非阻塞的方式打开文件，默认是阻塞
   	eg:以只读的方式打开已经存在的文件1.txt
   		int fd = open("1.txt",O_RDONLY);
   	多个状态标记需要用到位或连接 |
       eg:打开一个已经存在的文件1.txt，并且以读写的方式打开，在文件的末尾写入数据
       	int fd = open("1.txt",O_RDWR|O_APPEND);
   	mode:指定创建文件的权限，当第二个参数有O_CREAT才会有第三个参数
   		创建新的文件的权限 = mode & ~mask
   		有两种指定权限的方式
   		方式1：八进制
   			0777 
   		方式2：宏
           	
   返回值：
   	成功：返回一个打开文件的文件描述符 >2
   	失败：-1，同时设置错误码
   ```

   ```c

   参数：
   	fd:读取的那个文件的描述符

   	成功，返回实际读取到的字节数(<=count)

   ```

6. **lseek**

   ```c
   #include <sys/types.h>
   #include <unistd.h>
   
   off_t lseek(int fd, off_t  offset,  int  whence);
   功能：重新定位文件偏移量(光标)
   参数
   	fd:要进行重新定位光标的那个文件的fd
   	offset:偏移量，数字，具体的含义要结合第三个参数
   	whence：定位的方式，从哪个位置开始偏移
   		SEEK_SET:文件开头
   		SEEK_END:文件末尾
   		SEEK_CUR:当前光标的位置
   返回值
   	成功：返回新光标距离文件开头的字节数
   	失败：-1，error被设置
   eg:
   	//移动光标到文件开头
       lseek(fd,0,SEEK_SET);
   ```

# 4.文件标准IO

## **1. 标准 IO 是什么**

- 标准 IO 是 C 标准库提供的输入输出接口，头文件是 `stdio.h`
- 标准 IO 的核心对象是 `FILE *`
- 标准 IO 底层通常还是会调用系统 IO
- 标准 IO 的特点是带缓冲，使用更方便，很多场景下效率更高

**2. 标准 IO 和系统 IO 的区别**

| 对比项     | 标准 IO                               | 系统 IO                          |
| ---------- | ------------------------------------- | -------------------------------- |
| 头文件     | `stdio.h`                             | `unistd.h`、`fcntl.h`            |
| 操作对象   | `FILE *`                              | 文件描述符 `int fd`              |
| 典型函数   | `fopen`、`fclose`、`fgets`、`fprintf` | `open`、`close`、`read`、`write` |
| 是否带缓冲 | 带缓冲                                | 一般不由 C 库提供缓冲            |
| 使用难度   | 更方便                                | 更底层                           |

**3. 什么是流**

- 流就是数据传输的抽象
- 标准 IO 把文件、键盘、屏幕等统一看成“流”
- 常见标准流：
| 名称     | 含义       | 默认设备 |
| -------- | ---------- | -------- |
| `stdin`  | 标准输入流 | 键盘     |
| `stdout` | 标准输出流 | 屏幕     |
| `stderr` | 标准错误流 | 屏幕     |

**4. 缓冲区**

- 标准 IO 带缓冲区，数据通常先进入缓冲区，再真正写入文件或设备
- 使用缓冲区的目的是减少系统调用次数，提高效率

| 缓冲类型 | 特点                         |
| -------- | ---------------------------- |
| 行缓冲   | 遇到换行 `\n` 或缓冲满时刷新 |
| 全缓冲   | 缓冲区满时才刷新             |
| 无缓冲   | 数据尽快立即输出             |

## **2.常见标准IO函数**

**5. 文件流的打开和关闭**

函数原型：

```c
FILE *fopen(const char *pathname, const char *mode);
int fclose(FILE *stream);
```

`fopen` 参数说明：

- `pathname`：文件路径
- `mode`：打开方式

常见打开方式：

| 模式   | 含义                       |
| ------ | -------------------------- |
| `"r"`  | 只读，文件必须存在         |
| `"r+"` | 读写，文件必须存在         |
| `"w"`  | 只写，不存在创建，存在清空 |
| `"w+"` | 读写，不存在创建，存在清空 |
| `"a"`  | 追加写，不存在创建         |
| `"a+"` | 读写追加，不存在创建       |
| `"rb"` | 二进制只读                 |
| `"wb"` | 二进制只写                 |
| `"ab"` | 二进制追加                 |

注意：

- `fopen` 成功返回 `FILE *`
- `fopen` 失败返回 `NULL`
- `fclose` 成功返回 `0`
- `fclose` 失败返回 `EOF`

常见写法：

```c
FILE *fp = fopen("1.txt", "r");
if (fp == NULL)
{
    perror("fopen");
}
```

函数：

作用：

- `fputc`：向文件中写一个字符

返回值：

- 成功返回字符对应的 ASCII 值
- 失败或结束返回 `EOF`

注意：

- 接收 `fgetc` 返回值建议用 `int`
- 因为要区分普通字符和 `EOF`

函数：

作用：

- `fgets`：从文件中读一行
- `fputs`：向文件中写一个字符串

参数说明：

- `s`：保存字符串的数组
- `n`：最多读取 `n-1` 个字符
- `stream`：文件流

返回值：

- `fgets` 成功返回 `s`，失败返回 `NULL`
- `fputs` 成功返回非负值，失败返回 `EOF`

注意：

- `fputs` 不会自动加换行
- `puts` 会自动加换行，但主要用于标准输出

去掉 `fgets` 读入的换行常用写法：

```c
name[strcspn(name, "\n")] = '\0';
```

## **4. 文件标准 IO 常见易错点**

- `fopen` 后忘记判空
- 用 `"w"` 打开文件导致原文件被清空
- `fclose` 后继续使用文件指针
- `fgetc` 返回值用 `char` 接，导致无法正确判断 `EOF`
- `fgets` 读入后忘记处理末尾 `\n`
- `fputs` 不会自动加换行
- `fscanf` 格式和文件内容不一致
- `%s` 不限制长度，容易越界
- 用 `while (!feof(fp))` 读文件
- 文本文件和二进制文件读写方式混用


按结构体读二进制：

## 2、GY39

有两种工作模式

```
1.只获取光照强度
2.只获取温湿度大气压强海拔
```

由外部控制器发送至 GY-39 模块（十六进制）

所有串口指令格式，帧头：0xa5 

指令格式：帧头+指令+校验和(8bit)  各占一个字节 共三个字节

指令：

10000001=>0x81 获取光照强度

10000010=>0x82 获取温湿度大气压强海拔

校验和(前面的数据累加的和，只会保留低8位)

0xa5+0x81 = 0x26

0xa5+0x82 = 0x27

char cmd[3] = {0xa5,0x81,0x26};

write(fd,cmd,3);

char cmd1[3] = {0xa5,0x82,0x27};



如果是第一种模式：

```c
接收数据，接收9个字节
char buf1[9] = {0};
read(fd,buf1,9);

buf[0] = 0x5a
buf[1] = 0x5a
buf[2] = 0x15

光照：
	Lux=(buf[4]<<24) | (buf[5]<<16) | (buf[6]<<8) | buf[7];
	Lux=Lux/100; (lux)
```

如果是第二种模式：

```
接收数据，接收15个字节
char buf2[15] = {0};
read(fd,buf2,15);
buf2[0] = 0x5a
buf2[1] = 0x5a
buf2[2] = 0x45

温度：
	T=(buf2[4]<<8)|buf2[5]; 
 	T=T/100 单位℃
气压：Byte6~Byte9 
     P=(buf2[6]<<24) | (buf2[7]<<16) | (buf2[8]<<8) | buf2[9] 
     P=P/100 单位 pa 
湿度：Byte10~Byte11 
     Hum=(buf2[10]<<8)|buf2[11]
     Hum=Hum/100 百分制
海拔：Byte12~Byte13
 	H=(buf2[12]<<8)|buf2[13] 单位 m
```

当用户对按钮进行按下 松开 单击等才做的时候，都会触发相关的事件，如果关心某个事件，订阅的时候把这个事件和一个函数关联起来，后续发生该事件，就会自动调用该函数，这个函数叫做该事件的回调函数

```c
lv_event_dsc_t * lv_obj_add_event_cb(lv_obj_t * obj, lv_event_cb_t event_cb, lv_event_code_t filter, void * user_data)
@obj：你需要添加的事件的对象名
@event_cb:回调函数，写你点击或松开的时候需要做的事件(函数名),类型 lv_event_cb_t
	typedef void (*lv_event_cb_t)(lv_event_t * e);
	回调函数的格式固定的
		void 函数名(lv_event_t * e)
		{
			//相关的代码
		}
	struct lv_event_t {
        void * current_target;//当前事件的对象
        void * original_target;//原始事件的对象
        lv_event_code_t code;//事件的编码 eg:松开 按下
        void * user_data;//添加事件的传入的数据
        ...
	};
@filter：触发的事件
typedef enum {
        LV_EVENT_ALL = 0,//所有事件
        LV_EVENT_PRESSED, //按下
        LV_EVENT_PRESSING, //正在被按下
        LV_EVENT_SHORT_CLICKED, //单击
        LV_EVENT_LONG_PRESSED, //长按
        LV_EVENT_CLICKED,  //点击
        LV_EVENT_RELEASED,//松开
	} lv_event_code_t;	
@user_data：用来传递用户数据给回调函数，如果不需要就为NULL
```

```c
static lv_point_t start_point;//按下
static lv_point_t end_point;//松开
/*
    0 点击
    1 上
    2 下
    3 左
    4 右
*/
static int get_slide(lv_point_t start_point,lv_point_t end_point)
{
    int x = end_point.x-start_point.x;
    int y = end_point.y-start_point.y;

    if(x == 0 && y == 0)
    {
        return 0;
    }
    else if(abs(x) > abs(y))//左和右
    {
        if(x>0)
        {
            return 4;
        }
        else
        {
            return 3;
        }
    }
    else //上和下
    {
        if(y>0)
        {
            return 2;
        }
        else
        {
            return 1;
        }
    }
}
static void test10_cb(lv_event_t *e)
{

    //获取事件编码
    lv_event_code_t code = lv_event_get_code(e);
    if(code == LV_EVENT_PRESSED)//按下
    {
        //获取一个点击坐标
        lv_indev_get_point(lv_indev_active(), &start_point);   
    }
    else if(code == LV_EVENT_RELEASED)//松开
    {
        //获取一个点击坐标
        lv_indev_get_point(lv_indev_active(), &end_point);

            printf("up\n");
        }

        }
        else if(r == 3)

        }

        }
    }
```
