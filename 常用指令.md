## 2.Linux常用指令

1. 文件与目录

   | **指令**    | **常用参数**                                  | **说明**                         |
   | ----------- | --------------------------------------------- | -------------------------------- |
   | **`ls`**    | `-l` (详细), `-a` (显示隐藏), `-h` (易读大小) | 列出目录内容                     |
   | **`cd`**    | `..` (上级), `~` (主目录), `-` (上次目录)     | 切换工作目录                     |
   | **`pwd`**   | 无                                            | 显示当前完整路径                 |
   | **`mkdir`** | `-p` (递归创建多级目录)                       | 创建新目录                       |
   | **`rm`**    | `-r` (递归删除目录), `-f` (强制删除)          | 删除文件或目录（**慎用 `-rf`**） |
   | **`cp`**    | `-r` (拷贝目录), `-p` (保留属性)              | 复制文件或目录                   |
   | **`mv`**    | 无                                            | 移动或重命名文件/目录            |
   | **`touch`** | 无                                            | 创建空文件或更新文件时间戳       |

2. 文件内容查看

   **`cat`**: 一次性显示文件全部内容。

   **`less`**: 分页查看长文件（支持上下滚动，按 `q` 退出）。

   **`head` / `tail`**:

   - `tail -n 20 file`: 查看最后 20 行。
   - **`tail -f file`**: **实时刷新**查看文件末尾（常用于看日志）。

   **`grep`**: 强大的文本搜索工具。

   - `grep "error" log.txt`: 在文件中搜索关键词。
   - `ps -ef | grep python`: 结合管道符过滤进程。

   **`find`**: 在文件系统中查找文件。

   - `find /path -name "*.conf"`: 按文件名搜索。

3. 系统状态与进程

   **`top` / `htop`**: 实时显示系统资源占用（CPU、内存）及进程列表。

   **`ps`**: 静态查看进程快照。

   - `ps -ef` 或 `ps aux`: 显示所有运行中的进程。

   **`kill`**: 终止进程。

   - `kill -9 <PID>`: 强制杀死指定进程。

   **`df`**: 查看磁盘剩余空间。

   - `df -h`: 以 GB/MB 格式显示。

   **`free`**: 查看内存使用情况。

   - `free -h`: 易读格式显示。

4. 网络相关

   **`ping`**: 测试网络连通性。

   **`ifconfig`** (或 `ip addr`): 查看本机的 IP 地址和网卡信息。

   **`netstat`**: 查看网络端口占用。

   - `netstat -tuln`: 查看当前系统监听的端口。

   **`curl` / `wget`**:

   - `curl -X GET http://example.com`: 发送网络请求。
   - `wget http://url/file.zip`: 下载文件。

5. 权限相关

   **`chmod`**: 修改文件权限。

   - `chmod 755 file`: 设置为 rwxr-xr-x。
   - `chmod +x script.sh`: 赋予执行权限。

   **`chown`**: 修改文件所有者。

   - `chown user:group file`: 更改属主和属组。

   **`sudo`**: 以超级用户（root）身份执行命令。

6. 压缩与归档

   **`tar`**:

   - 解压：`tar -zxvf file.tar.gz`
   - 压缩：`tar -zcvf output.tar.gz /path/to/dir`

   **`zip` / `unzip`**: 处理 .zip 格式文件。

## 4. 库文件 / Makefile / CMake 相关指令

1. 动态库 (Shared Library, .so) 
    核心原理：运行期加载，多程序共享，极大地节省开发板内存和磁盘空间。

  ```makefile
  # 1. 将多个源文件编译为动态库 (必须加 -fPIC 和 -shared)
  # 规范：输出文件必须以 lib 开头，以 .so 结尾
  gcc -shared -fPIC bst.c linkqueue.c -o libbst.so
  
  # 2. 编译主程序并链接动态库
  # -I ./ : 告诉编译器去当前目录找头文件 (如 bst.h)
  # -L ./ : 告诉链接器去当前目录找库文件
  # -l bst : 链接库，掐头去尾 (去lib，去.so)
  gcc main.c -I ./ -L ./ -l bst -o app
  
  # 3. 运行前设置环境变量 (给操作系统动态加载器指路)
  export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:.
  ./app
  ```

  

2. 静态库 (Static Library, .a)
    核心原理：编译期直接将机器码拷贝进可执行文件中，体积大，但不依赖外界环境。

  ```makefile
  # 1. 先生成目标文件 (.o)
  gcc -c bst.c linkqueue.c
  
  # 2. 使用 ar 工具打包成静态库 (必须以 lib 开头，.a 结尾)
  ar rcs libbst.a bst.o linkqueue.o
  
  # 3. 链接主程序 (命令与动态库完全一样，GCC优先链接同名动态库)
  gcc main.c -I ./ -L ./ -l bst -o app_static
  ```

  

3. makefile

  核心痛点解决：替代手动敲击 gcc 的繁琐过程，实现增量编译（只编译修改过的文件）。

  ```makefile
  # ==========================================
  # 1. 变量定义区 (方便一键切换为交叉编译)
  # ==========================================
  # 如果要在开发板跑，把 gcc 改成 aarch64-linux-gnu-gcc
  CC = gcc  
  CFLAGS = -I./ -fPIC -Wall
  LDFLAGS = -L./ -lbst
  
  # ==========================================
  # 2. 终极目标区
  # ==========================================
  all: libbst.so app
  
  # ==========================================
  # 3. 规则定义区
  # 魔法变量速查：
  # $@ : 代表目标文件 (冒号左边)
  # $^ : 代表所有依赖文件 (冒号右边)
  # $< : 代表第一个依赖文件
  # ==========================================
  
  # 目标：生成动态库
  libbst.so: bst.c linkqueue.c
  	$(CC) -shared $(CFLAGS) -o $@ $^
  
  # 目标：生成可执行程序
  app: main.c libbst.so
  	$(CC) $< $(CFLAGS) $(LDFLAGS) -o $@
  
  # ==========================================
  # 4. 伪目标与清理
  # ==========================================
  .PHONY: clean
  clean:
  	rm -f app libbst.so *.o
  ```

  



1. 核心 CMakeLists.txt 模板
    无论工程怎么变，最核心的骨架只有这 6 句代码
    
    ```cmake
    # 1. 设定版本与项目名称
    cmake_minimum_required(VERSION 3.10)
    project(RK3568_Project)
    
    # 2. 指定头文件搜索路径 (等价于 -I)
    include_directories(${CMAKE_CURRENT_SOURCE_DIR})
    
    # 3. 生成动态库 (等价于 gcc -shared -fPIC...)
    # CMake 会自动补充 lib 前缀和 .so 后缀
    add_library(bst SHARED bst.c linkqueue.c)
    
    # 4. 生成可执行文件
    add_executable(app main.c)
    
    # 5. 链接依赖库 (等价于 -L 和 -l)
    # CMake 非常聪明，它知道 bst 在哪，全自动处理链接路径
    target_link_libraries(app bst)
    ```

3. Make执行指令注意

    ```bash
    make (-n)或(-s)
    	make首先在当前目录下面找Makefile/makefile,然后在该makefile中找第一个目标，执行
    
    make (-n)或(-s) 目标名
    	make首先在当前目录下面找Makefile/makefile,然后在该makefile中找目标，再去找依赖文件....再去执行达成目标命令列表
    	
    make -f makefile所在的路径名 目标名 	=>makefile的名字可以任意起
    	make首先会在makefile所在的路径去查找，然后再到makefile这个文件找 目标名 执行
    
    -n	只打印执行的命令，不显示结果
    -s	执行命令显示结果，但是不显示执行的命令
    ```

  

4. 内置函数

   ```makefile
   1. wildcard（通配符自动扫描）
   作用：像雷达一样，去指定的目录扫描所有符合规则的文件名，并用空格隔开。
   语法：$(wildcard 匹配模式)
   
   # 获取当前目录下所有的 .c 文件，存入变量 SRC
   SRC = $(wildcard *.c)
   
   2. patsubst（模式批量替换）
   作用：把一串文本中符合条件的单词，批量替换成你想要的格式（常用于把 .c 列表映射为 .o 列表）。
   语法：$(patsubst 查找模式, 替换模式, 目标文本串)
   # 把 SRC 变量里所有的 .c 替换成 .o，存入变量 OBJ
   OBJ = $(patsubst %.c, %.o, $(SRC))
   
   3. shell（执行外部系统命令）—— [高频补充]
   作用：在 Makefile 里直接运行 Linux 终端命令，并把结果拿回来。
   语法：$(shell 系统命令)
   # 获取当前所在的绝对路径
   CUR_DIR = $(shell pwd)
   # 获取系统的架构信息 (用于判断是不是 aarch64)
   ARCH = $(shell uname -m)
   
   4. notdir（剥离目录路径）—— [高频补充]
   作用：当你拿到的文件名带有很长的路径时，用它可以只保留最后的文件名。
   语法：$(notdir 包含路径的文件名列表)
   # 去掉 src/ 路径，只保留文件名
   FILES = $(notdir src/main.c src/add.c)
   # 此时 FILES 的值就是 "main.c add.c"
   ```

   

## 5.Makefile万能模板

1. Makefile万能模板

   ```makefile
   # 1. 编译器与目标名称定义
   # 【注意】如果你要放到 RK3568 开发板上跑，请将 gcc 改为 aarch64-linux-gnu-gcc
   CC = gcc
   #CC = aarch64-linux-gnu-gcc
   TARGET = app
   
   # 2. 编译选项与链接路径配置
   CFLAGS = -I./ -Wall -g      # -I 指定头文件路径，-Wall 开启警告，-g 支持GDB调试
   LDFLAGS = -L./              # -L 指定库文件路径 (如果需要链动态库，在后面加 -lxxx)
   LIBname = -l
   
   # 3. 魔法函数：自动处理源文件
   # 扫描当前目录下所有的 .c 文件
   SRC = $(wildcard *.c)
   # 将所有的 .c 文件名映射为对应的 .o 文件名
   OBJ = $(patsubst %.c, %.o, $(SRC))
   
   # 4. 终极编译规则 (链接成可执行文件)
   $(TARGET): $(OBJ)
   	$(CC) $^ -o $@ $(LDFLAGS)
   
   # 5. 模式编译规则 (教 make 如何把 .c 编译成 .o)
   %.o: %.c
   	$(CC) -c $< -o $@ $(CFLAGS)
   
   # 6. 防坑清理规则
   .PHONY: clean
   clean:
   	rm -f $(TARGET) $(OBJ)
   ```

   

2. Cmake万能模板

   ```cmake
   # 1. 指定 CMake 的最低版本要求
   cmake_minimum_required(VERSION 3.10)
   
   # 2. 设置编译路径（指定工具链）
   set(CMAKE_C_COMPILER /usr/local/arm/gcc-linaro-7.5.0-2019.12-x86_64_aarch64-linux-gnu/bin/aarch64-linux-gnu-gcc)
   set(CMAKE_CXX_COMPILER /usr/local/arm/gcc-linaro-7.5.0-2019.12-x86_64_aarch64-linux-gnu/bin/aarch64-linux-gnu-g++)
   
   # 2. 定义项目名称 (后续可以通过 ${PROJECT_NAME} 变量来引用它)
   project(RK3568_App)
   
   # 3. 设置全局编译选项 (相当于 Makefile 里的 CFLAGS)
   # 这里开启了所有警告，并加入了 GDB 调试信息支持
   add_compile_options(-Wall -g)
   
   # 4. 指定头文件的搜索目录 (相当于 Makefile 里的 -I ./)
   include_directories(${CMAKE_CURRENT_SOURCE_DIR})
   
   # 5. 魔法扫描：自动查找当前目录下所有的源码文件，并存入变量 DIR_SRCS 中
   aux_source_directory(. DIR_SRCS)
   
   # 6. 指定库文件所在的路径
   link_directories(./lib)
   
   # 7. 添加lib子目录
   add_subdirectory(./lib)
   
   # 8. 生成可执行文件
   # 使用上一步扫描到的所有源码，生成以项目名字命名的可执行文件
   add_executable(${PROJECT_NAME} ${DIR_SRCS})
   
   # 9. 链接依赖的动态库/静态库 (相当于 Makefile 里的 -L 和 -l)
   # 【注意】如果你的项目不需要链接额外的库，这一行可以直接注释掉
   target_link_libraries(${PROJECT_NAME} pthread m)
   ```
