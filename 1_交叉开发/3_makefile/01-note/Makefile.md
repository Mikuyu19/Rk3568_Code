```
管理项目的工具
	解决哪些文件先编译哪些文件后编译的文件

1.大量代码的关系的维护
	大项目源代码比较多，手工的维护，编译的时间长并且编译代码命令复杂，难以记忆和维护
2.减少重复编译的时间
	改动其中的一个文件，makefile能够判断哪些文件被修改过的，只对该文件进行重新编译，然后重新链接所有的文件，节省编译的时间

```

```
sudo apt install make
```

Makefile文件名

```
makefile或Makefile,推荐使用Makefile
```

Makefile格式

```
一个最简单的Makefile由很多的规则以及规则的指令组成
```

# 1.Makefile语法

一条规则

```
目标:依赖文件列表
命令列表
```

makefile的三要素

```
1.目标：通常是一个要生成的文件名或为你要执行的动作起的名字
2.依赖文件(也可以没有依赖文件)
3.命令
	可以由很多个命令，每个命令占一行(也可以没有命令)
```

# 2.make命令的使用

```c
make (-n)或(-s)
	make首先在当前目录下面找Makefile/makefile,然后在该makefile中找第一个目标，执行

make (-n)或(-s) 目标名
	make首先在当前目录下面找Makefile/makefile,然后在该makefile中找目标，再去找依赖文件....再去执行达成目标命令列表
	
make -f makefile所在的路径名 目标名 	=>makefile的名字可以任意起
	make首先会在makefile所在的路径去查找，然后再到makefile这个文件找 目标名 执行


-n	只打印执行的命令，不显示结果
-s	执行命令显示结果，但是不显示执行的命令
```

练习：为你的二叉排序树的代码main.c bst.c bst.h linkqueue.c linkqueue.h写一个makefile 编译make生成main可执行文件

# 3.makefile变量

```
没有类型的概念，当成字符串，变量可以以数字开头
```

引用变量值

```
$(变量名)
```

在makefile给变量复制，有4种情况

```c
1.
	:= 就地复制 不会向后展开
	
2.
	= 递归赋值 向后展开

3.
	+= 追加赋值

4.
	?=	条件赋值
```

自动变量

```
内置变量，早就有人帮定义好了，直接使用

$@	目标文件的完整的名称
$^	所有的不重复的依赖文件的文件名，以空格隔开
$<	第一个依赖文件的名字

注意：自动变量只能在规则命令中使用
```

# 4.make内置函数

```
makefile运行定义调用函数，在makefile中，调用内置的函数，自定义的函数
	调用函数格式
		$(函数名 函数参数列表) //函数参数列表用逗号隔开

make也运行调用shell命令或脚本，在makefile中，调用shell命令的格式
	$(shell 命令或脚本 参数)
```

```
wildcard函数
	文件名展开函数，展开成符合参数描述的文件，展开后文件名以空格隔开
	只带一个参数：指定文件名的通配方式

patsubst函数
	模式字符串替换函数
	$(patsubst <form>,<to>,<text>)
	把原始字符串<text>中<form>替换<to>
	form:要替换的东西
	to:需要被替换成什么
	text：要替换的字符串
```
